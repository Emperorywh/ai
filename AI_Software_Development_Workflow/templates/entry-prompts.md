# 三段式可复用提示词（Entry Prompts）

> 这是「一句话驱动整个功能开发」的入口。三个提示词分别对应**访谈 → 规划 → 推进**三个阶段，每个都靠 `docs/FEATURES.md` 的「当前活动功能」指针定位。
>
> - ① 访谈、② 规划：每个功能各用一次。
> - ③ 推进：**每次粘贴同一句**，做完一个 Task 即停，等人工验收后再粘同一句做下一个。
>
> 用法：把对应「👇 复制这段」框内全文粘成消息即可。若项目根有 `CLAUDE.md`（自动加载），也可直接用触发词（访谈 / 规划 / 推进 或 继续·next）。

---

## ① 访谈提示词（开新功能 / 阶段 1）

```
按 CLAUDE.md 阶段 1 协议，对我要做的功能做深度访谈。

我要做的功能（初步描述）：<在这里写一句话需求，或贴出已有想法>

要求：
- 不要直接给方案。先理解问题，再逐轮深挖。
- 每轮覆盖以下维度之一并轮转，主动挖我可能没想到的难点：
  1) 技术实现（技术栈/架构/数据/接口/外部依赖）
  2) UI / UX（交互流程/关键界面/状态/异常态）
  3) 边界与异常（空/极值/并发/失败/权限/兼容）
  4) 顾虑与权衡取舍（成本/性能/安全/工期/可维护性，A vs B 的取舍）
  5) 隐性难点（容易被低估的复杂度、返工风险点）
- 每轮问完先小结你目前的理解、标记未决项，再继续追问；带默认值的决策给出【默认】建议。
- 持续访谈，直到：目标一句话能说清、MVP 边界明确、无阻塞开发的待确认项、决策日志齐全。
- 全部覆盖后，把完整规格写入 docs/features/<功能名>/SPEC.md（用 spec 模板，含决策日志/边界清单/风险权衡/待确认默认值），并在 docs/FEATURES.md 注册该功能、设为「当前活动功能」。
- 落盘后停下，等我确认 SPEC，不要自动进入规划。
```

---

## ② 规划提示词（阶段 2）

```
按 CLAUDE.md 阶段 2 协议，根据当前活动功能的 SPEC，拆分为「可独立验证的」Milestone 与 Task。

要求：
- 读 docs/FEATURES.md 取「当前活动功能」，读其 docs/features/<功能名>/SPEC.md。
- Milestone 按用户价值拆（纵向切片，可独立运行/测试/验收），高风险前置。
- Task 拆到「一轮 agent 工作可做完 + 自测通过」粒度，构成 DAG 无环；每个 Task 标风险等级（🟢🟡🔴）与依赖。
- 产出两份文件（用模板）：
  1) docs/features/<功能名>/ROADMAP.md：路线图 + 每 Milestone 详细设计（目标/价值/范围/不含/涉及目录/依赖/风险验证项/验收标准/DoD/「Claude Code 开发边界：允许·禁止目录」）+ Task 清单表（内联）+ 实施风险报告。
  2) docs/features/<功能名>/PROGRESS.md：初始化进度表（Task 全 ⬜、「当前指针」指向 Task 01）。
- 输出推荐开发顺序，然后停下等我确认规划，不要自动开始推进。
```

## ③ 推进提示词（阶段 3，每次粘贴同一句）

```
推进当前活动功能的一个 Task。做完即停，等我人工验收。

按序执行，不跳步：
1. 读 docs/FEATURES.md → 取「当前活动功能」及其目录 docs/features/<功能名>/。
2. 读该功能 PROGRESS.md → 「当前指针」定位「本次 Task = Task 状态表第一个非 ✅」。若全部 ✅，告诉我并停下。
3. 读该功能 ROADMAP.md 该 Task 行 + 所属 Milestone 的「开发边界 / 验收标准」；读 SPEC.md 相关章节（技术栈 / 构建命令 / 共享契约从 SPEC/ROADMAP 取，不要自己假设）。
4. 跑 git log --oneline -10，校验 PROGRESS 中 ✅ 的 Task 都有对应 commit；不一致以 PROGRESS 为准，但先告诉我。
5. 简报：【进度】【本次 Task】【做什么】【边界：允许改·禁止改】【验收】【依赖】。涉及架构/公共接口/数据格式/跨边界改动，先等我确认再动手。
6. 按该 Task 风险等级行动：🟢低/🟡中自主完成并自测（🟡完成后汇报改了什么）；🔴高先出实现方案等我确认。只做本 Task，YAGNI，只改边界内目录。
7. 写功能同时写测试，亲自跑测试/lint/typecheck 到全绿（不要只给测试建议）。
8. 更新 PROGRESS.md：该行 → ✅ + commit 短 hash + 备注；「当前指针」→ 下一个 Task；「近期注意事项」倒序追加踩坑/决策/验收结论。
9. commit（含 PROGRESS 更新），message 用简体中文。
10. 报告并停下：做了什么 / 自测与验收结果 / 下一个 Task 是什么 / 交我人工验收。不要自动做下一个 Task。
```

---

## ④ 切换活动功能（可选）

```
把 docs/FEATURES.md 的「当前活动功能」指针切到 <功能名>，然后报告该功能当前进度（当前 Milestone / Task / 总体进度）。之后用「推进」即可继续它。
```
