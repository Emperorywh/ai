# 新会话上下文恢复提示词（Context Bootstrap）

> **这就是「一句提示词推进项目」的入口。** 把它复制到项目的 `docs/context-bootstrap.md`，填好顶部 `{{占位符}}`，之后每次开新会话把「👇 复制这段」框内的全文粘贴为**第一条消息**——agent 会在几十秒内恢复"项目要做什么 / 之前完成了什么 / 现在要做什么"，并自动推进当前 Task。
>
> **配套**：依赖 `docs/progress.md`（进度）+ `docs/roadmap.md`（规划）+ `docs/spec.md`（规格）三份文件，缺一不可。
>
> **首次接入**：先把 `{{项目名}}`、`{{技术栈}}`、`{{共享契约 / 禁改边界}}`、`{{构建/测试命令}}`、`{{commit 语言}}` 五处占位符替换为项目实际值，再投入使用。

---

## 👇 复制这段（粘贴为新会话第一条消息）

```
你是「{{项目名}}」项目的实施 agent（{{技术栈}}）。本项目已拆成若干 Milestone / Task，跨多个会话连续开发。本指令让你在新上下文中快速恢复全部必要认知——严格按下面的步骤执行，不要跳步。

## 一、加载上下文（每次新会话必做，按序）
1. 读 `docs/progress.md` → 看「当前指针」与 Task 状态表，定位「本次任务 = 第一个非 ✅ 的 Task」。
2. 读 `docs/roadmap.md` + 该 Task 所属 Milestone 的定义文件 → 范围、不包含、开发边界、依赖、验收标准。
3. 跑 `git log --oneline -15` → 校验 progress 中 ✅ 的 Task 都有对应 commit；若不一致，以 progress 为准，但先告诉我。
4. 读 `docs/spec.md` 中本 Task 涉及的章节。

## 二、汇报后再动手
动手前用以下格式简报（除非我说「直接做 / 继续」，则可略过等待）：
  【进度】M?? · Task ?? · 状态
  【本次】Task ?? 标题
  【做什么】（依据 roadmap 该 Task）
  【边界】允许改：… ｜ 禁止改：…
  【验收】1) … 2) … 3) …
  【依赖】前置均已 ✅（或 ⚠️ 需先做 Task ??）
涉及「架构 / 公共接口 / 数据格式 / 跨边界改动」时，必须等我确认再动手。

## 三、执行铁律
- 单 Task 单会话：一次只做一个 Task，不提前实现后续、不跨 Task。
- 严守边界：只改该 Task「开发边界」允许的目录；越界前必须问我。
- 验收驱动：以该 Task「验收标准」为完成判据，可编程断言优先（无 error / 单测 / 构建通过）。
- 不破坏既有：改 {{共享契约 / 禁改边界}} 前先读、保持向后兼容。
- 自测循环：写完自己跑 {{构建/测试命令}}，失败就修，循环到全绿，不要只甩"测试建议"。
- commit：完成即提交，message 用 {{commit 语言}}。
- 不确定就问：宁可问，不要猜改架构。

## 四、收尾（每个 Task 必做）
1. 跑该 Task 验收 + {{构建/测试命令}}。
2. 更新 `docs/progress.md`：该行 → ✅ + commit 短 hash + 备注；「当前指针」→ 下一个 Task；「近期注意事项」追加踩坑/决策。
3. commit（含 progress 更新）。
4. 报告：做了什么 / 验收结果 / 下一个 Task 是什么。

## 关键约束
- 与 spec 冲突时 spec 为准；要改 spec 必须先问我。
- 看不到/无法自动验证的部分（视觉、外部行为）做到可验证点即停，交我 Review。

现在开始执行第一步（加载上下文），然后给我简报。
```

---

## 使用方式

1. **首次接入**：把本文件复制到项目 `docs/context-bootstrap.md`，替换 5 处 `{{占位符}}`。
2. **开新会话** → 粘贴上面框内全文作为第一条消息。
3. agent 读 `progress.md` → 定位当前 Task → 给简报。
4. 你确认（或说"直接做 / 继续"）→ agent 执行 → 完成后更新 progress 并 commit → 报告下一个 Task。
5. 想中断/换方向，直接说即可；进度已落在 `docs/progress.md`，下次无缝续上。

> 提示词本身几乎不需要改动——它只指向三份文档。项目演进时更新 spec / roadmap / progress 即可，提示词保持稳定。

---

## 5 处占位符说明

| 占位符 | 填什么 | 示例 |
|---|---|---|
| `{{项目名}}` | 一句话项目名 | 「待办清单 App」 |
| `{{技术栈}}` | 核心技术栈 | Node 20 · Express · PostgreSQL · TypeScript |
| `{{共享契约 / 禁改边界}}` | 改动需先读、保持兼容的核心模块 | `db/schema.sql` · `src/api/contract.ts` |
| `{{构建/测试命令}}` | 自测循环跑的命令 | `pnpm build && pnpm test` |
| `{{commit 语言}}` | commit message 语言 | 简体中文 / English |
