## 维护规则（给 agent）

1. **每完成一个 Task**：该行 → ✅ + commit 短 hash + 备注；「当前指针」→ 下一个 Task；「近期注意事项」追加经验。
2. **开始一个 Task**：该行 → 🔄（in_progress）；开始前 `git log` 校验与本表一致。
3. **遇阻**：状态 → ⚠️ blocked，备注写阻塞原因与所需输入，**停下问我**。
4. **绝不**跳过未 ✅ 的前置 Task；**绝不**改本文件来「绕过」依赖。
5. **做完单个 Task 即停**，更新本文件并 commit，报告后等用户人工验收，不自动做下一个 Task。
