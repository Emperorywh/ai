> 三段式提示词（访谈 / 规划 / 推进）都靠这里的指针定位「现在做哪个功能」。
> 复制本文件到项目 `docs/FEATURES.md` 后维护。

| 项 | 值 |
|---|---|
| 文档版本 | v1.0 |
| 维护者 | Claude Code（开新功能 / 切换功能时更新）+ 人工 |

## 当前活动功能

> **agent 推进开发时读取的唯一指针。** 同一时刻只指向一个功能；要换功能就改这里（或直接说「切到 <功能名>」）。

- **当前活动功能**：`<功能名>`
- **文档目录**：`docs/features/<功能名>/`
- **当前阶段**：访谈中 / 规划中 / 开发中（当前 Milestone / Task 见该功能 `PROGRESS.md`）

## 维护规则（给 agent）

1. **新开功能（阶段 1 访谈结束落盘 SPEC 时）**：在清单新增一行；把「当前活动功能」指针指到它。
2. **切换功能**：只改「当前活动功能」与「文档目录」两行；不清空其他功能的进度。
3. **状态联动**：各功能的 SPEC/ROADMAP/PROGRESS 是否就绪，以该功能目录下的实际文件为准，本表只做总览；每次推进后可顺手刷新「Milestone 进度」与状态。
4. **功能发布（阶段 5）**：把该行状态改为 ✅已发布。
